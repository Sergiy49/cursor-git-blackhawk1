gitignore потрібен для приховування файлів і папок від системи контролю версій Git. Зазвичай приховують конфігураційні файли (особливо з паролями), тимчасові файли і папки. gitignore використовує glob формат для вибірки файлів.

git config --global user.name blabla-використовуєтся щоб встановити ім’я користувача

git config --global user.email blabal@bla-використовуєтс щоб встановити email користувача

git init-використовуєтся для створення нового репозиторія з каталогу проект

git status-використовуєтся для перевірити, чи є у вас зміни

git add-використовуєтся для зберігання змін

git commit-використовуєтся для створення комітів

git log-історія комітів

git checkout-використовуєтся для повернення робочого каталогу до будь-якого попереднього стану

git branch-використовуєтся для створення нової гілки

git push-використовуєтся для надсилання змін на віддалений репозиторій

git pull-використовуєтся для вивільнення змін з віддаленого репозиторію

git merge-використовуєтся для зливання двох гілок

git clone-використовуєтся для отримання копії існуючого репозиторію

git revert-дозволяє скасувати будь-які коміти у репозиторію

git reset-використовуєтся для скасування індексації змін

git rebase-всі зміни, що були в комітах одної гілки та відтворити їх на іншій гілці.

git cherry-pick-Команда git cherry-pick використовується для того щоб взяти зміни, внесені будь-яким коммітов, і спробувати застосувати їх заново у вигляді нового коммітов нагорі поточної гілки. Це може виявитися корисним щоб забрати парочку коммітов з іншої гілки без повного злиття з тією гілкою

git stash-Ховання (stashing) бере чорновий стан вашої робочої директорії — тобто ваші змінені супроводжувані файли та індексовані зміни — та зберігає їх у стеку незавершених змін, які ви можете знову використати будь-коли (і навіть на іншій гілці)
